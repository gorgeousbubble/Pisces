# 实现任务列表

## 任务概览

本项目分为两大部分：**MCU 固件**（C语言，NXP KSDK + FreeRTOS）和**树莓派服务器**（Python，FastAPI）。任务按依赖关系排序，可并行执行的任务已标注。

---

## 阶段一：MCU 固件基础层

### 任务 1：搭建 MCU 固件工程脚手架
- [ ] 1.1 在 MCUXpresso IDE 中创建 MK64FN1M0VLL12 工程，选择 KSDK 2.x
- [ ] 1.2 集成 FreeRTOS（KSDK 内置版本），配置堆大小为 64KB，最大任务数 10
- [ ] 1.3 集成 FatFs（R0.15），配置 SDHC 底层驱动接口
- [ ] 1.4 配置链接脚本，确认 Flash(1MB) 和 RAM(256KB) 分区正确
- [ ] 1.5 实现串口日志模块（UART0，115200bps），支持带时间戳的格式化输出
- [ ] 1.6 实现系统时钟初始化（120MHz，PLL配置）
- **需求关联**：需求 8（配置管理）、需求 9（可靠性）
- **验收**：工程可编译，串口输出 "System Init OK"，FreeRTOS 调度器正常启动

---

### 任务 2：实现配置文件读取模块
- [ ] 2.1 实现 `config_load()` 函数，从 SD 卡根目录读取 `config.ini`
- [ ] 2.2 实现 INI 格式解析器，支持 `[section]` 和 `key=value` 格式
- [ ] 2.3 实现参数合法性校验（WiFi SSID 非空、quality 50–95、fps 1–30）
- [ ] 2.4 实现默认配置回退逻辑，记录配置加载失败日志
- [ ] 2.5 定义全局配置结构体 `g_config`，供各模块读取
- **需求关联**：需求 8.1、需求 8.2
- **依赖**：任务 1（工程脚手架）、任务 4（SD卡驱动）
- **验收**：SD卡存在 config.ini 时正确加载；文件缺失时使用默认值并记录日志

---

### 任务 3：实现看门狗与系统管理器
- [ ] 3.1 初始化 K64 WDOG 模块，超时时间 5 秒，使能中断
- [ ] 3.2 实现 `task_watchdog` 任务（优先级最高），每 1 秒喂狗
- [ ] 3.3 实现各任务心跳标志位机制（`task_heartbeat[]` 数组）
- [ ] 3.4 实现 `sys_manager_task`：检查心跳、统计运行时间、维护 `sys_status_t`
- [ ] 3.5 实现软复位函数 `sys_soft_reset()`，记录复位原因到 Flash（掉电保持）
- [ ] 3.6 实现 Drop_Counter 原子读写接口
- **需求关联**：需求 9.1、需求 9.2
- **依赖**：任务 1
- **验收**：人为阻塞主循环 6 秒后系统自动复位；复位原因可从日志读取

---

## 阶段二：MCU 外设驱动层（可与阶段一并行）

### 任务 4：实现 SD 卡驱动与文件管理器
- [ ] 4.1 配置 K64 SDHC 控制器（4位模式，时钟 25MHz）
- [ ] 4.2 实现 FatFs 底层驱动接口（`disk_read`/`disk_write`/`disk_ioctl`）
- [ ] 4.3 实现 `fm_init()`：挂载 FAT32 文件系统，检测 SD 卡插入状态
- [ ] 4.4 实现 `fm_start_recording()` / `fm_stop_recording()`：创建/关闭 MJPEG 文件
- [ ] 4.5 实现 `fm_write_frame()`：写入4字节长度头 + JPEG数据
- [ ] 4.6 实现 `fm_save_snapshot()`：保存 JPEG 照片文件
- [ ] 4.7 实现 `fm_get_free_space()`：查询剩余空间
- [ ] 4.8 实现文件轮转逻辑：文件大小 ≥ 3.9GB 时自动新建文件
- [ ] 4.9 实现 SD 卡不可用时的降级处理（禁用存储，上报状态）
- **需求关联**：需求 3（全部）
- **依赖**：任务 1
- **验收**：SD卡写入速率 ≥ 4MB/s（用逻辑分析仪或计时验证）；文件命名格式正确

---

### 任务 5：实现摄像头驱动（OV2640）
- [ ] 5.1 实现 SCCB（I2C）驱动，配置 I2C0（400kHz）
- [ ] 5.2 实现 OV2640 寄存器初始化序列（VGA，JPEG输出，15fps）
- [ ] 5.3 配置 K64 DVP 接口（GPIO + DMA），接收 OV2640 JPEG 数据流
- [ ] 5.4 实现 VSYNC/HREF 中断处理，标记帧开始和结束
- [ ] 5.5 实现 DMA 传输完成回调，将帧数据指针放入 `frame_queue`
- [ ] 5.6 实现帧超时检测（66ms），超时则丢帧并 Drop_Counter++
- [ ] 5.7 实现 `cam_reinit()`：重新执行初始化序列
- [ ] 5.8 实现连续初始化失败计数（3次后停止重试）
- **需求关联**：需求 1（全部）、需求 2.1
- **依赖**：任务 1
- **验收**：摄像头输出稳定 15fps JPEG 帧；帧超时时 Drop_Counter 正确递增

---

### 任务 6：实现 JPEG 编码器配置模块
- [ ] 6.1 实现 `encoder_set_quality()`：将质量因子（50–95）映射并写入 OV2640 QS 寄存器
- [ ] 6.2 实现 `encoder_set_resolution()`：切换 OV2640 输出分辨率（VGA/720P）
- [ ] 6.3 实现拍照模式切换：临时切换至 1280×720，quality ≥ 80，采集单帧后恢复
- [ ] 6.4 实现编码超时检测（200ms），超时丢帧并记录日志
- [ ] 6.5 实现编码输出缓冲区满检测，丢帧并记录溢出事件
- **需求关联**：需求 2（全部）、需求 7.3
- **依赖**：任务 5
- **验收**：质量因子变更后输出 JPEG 文件大小变化符合预期；720P 拍照帧可正常解码

---

### 任务 7：实现 WiFi 网络传输层
- [ ] 7.1 实现 UART1 驱动（DMA收发，115200bps），用于 AT 指令通信
- [ ] 7.2 实现 AT 指令解析器（发送命令、等待响应、超时处理）
- [ ] 7.3 实现 `net_connect()`：AT+CWJAP 连接 WiFi，超时 30 秒
- [ ] 7.4 实现 WiFi 连接重试逻辑（间隔 10s，最多 5 次）
- [ ] 7.5 实现 TCP 连接建立（AT+CIPSTART），超时 30 秒
- [ ] 7.6 实现 MJPEG-over-HTTP 推流（AT+CIPSEND，透传模式）
- [ ] 7.7 实现 TCP 断线检测与重连（间隔 5s，最多 5 次）
- [ ] 7.8 实现 WiFi 模块硬件复位（拉低 RST 引脚 100ms）
- [ ] 7.9 实现 60 秒无连接触发硬件复位逻辑
- [ ] 7.10 实现命令接收通道（解析服务器下发的 JSON 命令）
- **需求关联**：需求 4（全部）、需求 7.2
- **依赖**：任务 1、任务 2
- **验收**：WiFi 连接成功后 TCP 推流可被 Wireshark 抓包验证；断线后自动重连

## 阶段三：MCU 应用层

### 任务 8：实现主任务调度与数据流管道
- [ ] 8.1 创建 `encoded_frame_queue`（深度 2，每帧最大 40KB）
- [ ] 8.2 实现 `task_cam_capture`：从摄像头获取帧，放入队列
- [ ] 8.3 实现 `task_net_send`：从队列取帧，调用 `net_send_frame()`
- [ ] 8.4 实现 `task_file_write`：从队列取帧，调用 `fm_write_frame()`
- [ ] 8.5 实现 `task_cmd_handler`：处理拍照命令，协调编码器和文件管理器
- [ ] 8.6 实现离线存储模式：WiFi 不可用时仅写 SD 卡，恢复后继续推流
- [ ] 8.7 实现低功耗待机模式：停止采集，降低时钟频率
- **需求关联**：需求 1–4、需求 7、需求 9
- **依赖**：任务 3–7
- **验收**：端到端数据流正常；离线模式切换不丢失 SD 卡数据

### 任务 9：实现状态上报模块
- [ ] 9.1 实现 `sys_report_status()`：将 `sys_status_t` 序列化为 JSON 并通过 TCP 发送
- [ ] 9.2 实现定时上报（每 10 秒上报一次状态）
- [ ] 9.3 实现错误事件即时上报（摄像头失败、SD 卡告警、WiFi 断线）
- [ ] 9.4 实现 SD 卡剩余空间低于 50MB 的告警上报
- **需求关联**：需求 3.5、需求 3.7、需求 3.8、需求 8.3
- **依赖**：任务 7、任务 8
- **验收**：服务器端可收到格式正确的状态 JSON；告警事件在触发后 1 秒内上报
